# Claude Code 懶人包製作規範（模板）

> 所有懶人包都必須按照此模板製作，確保一致的品質與使用體驗。

---

## 必備結構（每個懶人包都要有）

### 1. 標題與版本資訊

```markdown
# Claude Code 懶人包 #XX：[名稱]

> 版本：vX.X
> 更新日期：YYYY-MM-DD
> 對應影片：Claude基本功 EPXX
```

### 2. Frontmatter 規範

```yaml
---
title: "Claude Code 懶人包 #XX：[名稱]"
date: YYYY-MM-DD
type: 懶人包
version: vX.X
status: 初版（實作後更新） / 實測中 / 穩定版
tags:
  - Claude-Code
  - 懶人包
  - [相關標籤]
video: EPXX
---
```

### 3. 四大必備區塊

#### ① 這個懶人包會幫你做什麼？
- 用 3-5 個 bullet point 說明完成後的效果
- 讓使用者在 10 秒內決定要不要用

#### ② 先備條件
- 用 checkbox 列出所有前提條件
- 使用者一看就知道自己缺什麼

#### ③ 操作指令（給 Claude Code 讀的）
- 必須包含以下子區塊（順序不可更動）：

```
步驟零：環境檢查 ← 每個懶人包都要有
步驟一～N：實際操作步驟
```

#### ④ 如果安裝失敗，如何重來
- 提供完整的復原步驟
- 讓使用者能回到乾淨狀態重新執行

---

## 步驟零：環境檢查模板（直接複製使用）

```markdown
### 步驟零：環境檢查

> 請 Claude 在開始前，先自動確認以下所有項目。
> 如果有任何一項不符合，請先告知使用者問題所在，並引導解決後再繼續。
> **不要跳過任何一項檢查，不要假設環境正常。**

1. **確認作業系統**：執行系統指令確認是 Windows / macOS / Linux，
   後續所有指令請根據實際的作業系統選擇正確版本執行
2. **確認網路連線正常**
3. [根據懶人包需求新增檢查項目]

> 全部通過後，告知使用者：「環境檢查完成，所有條件都符合，開始執行。」
> 如果有不通過的項目，列出問題清單並逐一引導解決。
```

---

## 寫作規則

### 執行環境原則

- **預設所有操作都在 Claude Code 桌面版的 Code 分頁內完成**
- 使用者不需要另外打開 PowerShell、命令提示字元或終端機
- 只有在桌面版確實無法執行某個指令時，才引導使用者到終端機操作
- 進階使用者可自行選擇使用 Claude Code CLI 版本

### 給 Claude 的指令要明確

- ✅ 「執行 `git --version`，如果未安裝，引導使用者安裝」
- ❌ 「確認 Git 有裝好」（太模糊，Claude 可能跳過）

### 手動操作要標記

- 使用 🖐️ 標記所有需要使用者手動操作的地方
- 說明要做什麼、為什麼要手動、做完後怎麼確認

### 跨平台指令要分開寫

- 不同作業系統的指令用明確的標題區分
- 或在步驟零要求 Claude 自動判斷後選擇正確版本

### 錯誤處理要具體

- 每個步驟後面都要寫「如果失敗怎麼辦」
- 不要只說「請重試」，要給具體的排查方向

## 版本管理規則

| 版本 | 含義 |
|------|------|
| v0.1 | 初版，根據文件撰寫，尚未實測 |
| v0.2 | 加入環境檢查、復原機制等防禦性設計 |
| v0.3+ | 加入實作過程中遇到的問題與解法 |
| v1.0 | 經多人實測後的穩定版本 |

### 更新流程

1. 在 Obsidian 中更新懶人包內容
2. 同步推送到 GitHub repo
3. 更新 frontmatter 中的 version 和 date
4. 在更新紀錄表格中新增一行
